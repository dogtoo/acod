ACOD

一個可將網頁元素內容迅速收集或將response資料怖局回元素內的開發library，以jQuery為設計核心可配合使用者要求擴增更多功能。 主要將網頁元素設定特定標籤及劇本，Acod會歷遍tag後依照指定的劇本對元素做處理。方便開發者可省去畫面與後端的資料收集及配置。

acod 的結構

主要的程式有advconfig.js、advajax.js、advgrid.js、advpane.js，依需求配合擴增advdecl.js、advbtn.js、advassist.js。開發者在程式開頭需件js檔 案載入並宣告其定義。

        	        
<script src="js/advconfig.js"></script> //找到所有被註冊為ACOD的元件 
<script src="js/advajax.js"></script> //基於jQuery程式，依使用者設定情境達到基本的UI效果 
<script src="js/advgrid.js"></script> //進階的advajax，專門處理Grid的部份 
<script src="js/advpane.js"></script>
<script src="js/advdecl.js"></script>
<script src="js/advbtn.js"></script>
<script src="js/advassist.js"></script>

<script type="text/javascript">
    advConfigRegister();
    advPaneRegister();
    advAjaxRegister();
    advGridRegister();
    advDeclRegister();
</script>

ACOD的好處

透過advAjax將基本元素如input、textarea、select、label依照設定的劇本做取值設值的處理。

當元素過多時會造成程式撰寫的困難，若要處理response資料有關到table部份會更加複雜，因此我們提供advAjaxGetRequest及advAjaxSetResponse二個功能對元素做取值及設值的動作。 接下來我們來看如何時用這二個功能，還有我們提供了那些功能。

如何使用ACOD

設定Acod標籤

我們在網頁元素下加上<div class="acod" ></div> ，當網頁載入後會先執行advAjaxRegister();，該程式會歷遍所有元素並找到acod的標籤，並將標籤內的劇本記錄下來。

加入劇本

一個作業裡基本功能應該會有「查詢、新增、更新、刪除」，這些功能也就是我們所稱的劇本，主要目的在不同的劇本下應該會使用到不同 的元素，我們只需要收集或設定有必要的元素避免將太多不必要的資料送到後端或浪費瀏覽器效率於顯示資料上。

// 可預設查到二筆資料TOM與SAM
var I2_data =
    [
      { 'I2_name':'TOM'
      , 'I2_phone':'12345678'
      , 'I2_addr':'New 9RD Taipei'}

    , { 'I2_name':'SAM'
      , 'I2_phone':'87654321'
      , 'I2_addr':'No.2, Yongkang St Taipei'}
    ];


function I2_example(Scenario) {
    var request = advAjaxGetRequest(Scenario);
    if (Scenario == "query") {
    // 模擬後端查詢
        var query;
        for (var i = 0 ; i < I2_data.length ; i++)
        {
            if (I2_data[i] && I2_data[i].I2_name == request["I2_name"]) {
                query = I2_data[i];
            }
        }
    // 查詢END

    // 將查詢的資料顯示到畫面上
        if (query) {
            advAjaxSetResponse("query", query);
            advConfigShowErrorMessage("查詢成功", 0);
        }
        else
            advConfigShowErrorMessage("查無資料", -1);
    }
    else if (Scenario == "add") {
    // 模擬後端新增資料
        var i = I2_data.length;
        I2_data[i] = {'I2_name' : request["I2_name"], 'I2_phone' : request["I2_phone"], 'I2_addr' :request["I2_addr"]};
        advConfigShowErrorMessage("新增完成", 0);
    }
    else if (Scenario == "update") {
    // 模擬後端更新資料
        advConfigShowErrorMessage("無更新資料", -1);
        for (var i = 0 ; i < I2_data.length ; i++)
        {
            if (I2_data[i].I2_name == request["I2_name"]) {
                I2_data[i] = {'I2_name' : request["I2_name"], 'I2_phone' : request["I2_phone"], 'I2_addr' :request["I2_addr"]};
                advConfigShowErrorMessage("更新完成", 0);
                return false;
            }
        }
    }
    else if (Scenario == "delete") {
        for (var i = 0 ; i < I2_data.length ; i++)
        {
            if (I2_data[i] && I2_data[i].I2_name == request["I2_name"]) {
                delete I2_data[i];
                // 此時request={}，advAjax還是會依劇本去處理所有元件
                // 所以會將所有元件資料清空
                advAjaxSetResponse("delete", request);
                return false;
            }
        }
    }
}

劇本設定

提供3種request、response方向，決對定方向後填入劇本。

  • to 對 AP server 發出需求時，應送出此元件值或屬性
  • fr 當 AP server 回應結果時，應設定此元件值或屬性
  • bi 等同於 to + fr

    • 填入劇本後可為to:query;add; fr:delete或bi:query;

    執行語法

    收集劇本元素及配置資料至元素。

  • advAjaxGetRequest(Scenario): 收集所有設定劇本的元素。
  • advAjaxSetResponse(Scenario,response): 將資料(response)配置到劇本元素。response為後端程式處理後的資料。

    • 設定欄位屬性

    除了可以傳送欄位值外也可以傳欄位屬性到後端，或由後端控制前端畫面。

    範列：傳送及設定欄位

    bi:setRead(read);setSRead(read)
    bi:setRead

<input type="text" name="I3_name">
<div class="acod">bi:setRead(read)</div>
<label>設為欄位唯讀</label>

<div class="acod">bi:setRead</div>
<button type="button" onclick="I3_example('setRead')">設定或取消唯讀</button>

function I3_example(Scenario) {
    var request = advAjaxGetRequest(Scenario);
    var response = {};
    if (Scenario == "setRead") {
        response = request;
    }
    if (Scenario == "setSRead") {
        response["I3_nameRead"] = 1;
    }
    advAjaxSetResponse(Scenario, response);
}

    元素屬性設定

  • read: 自動產生{name}Read資料Request該元素是否為唯讀，Response將{name}Read為true時該元素設定為唯讀。
  • bgColor: 自動產生{name}BgColor資料Request該元素背景色RGB代碼。Response時為設定該元素背景色。
  • selText: 自動產生{name}SelText資料Request該元素選定值，Response時可預設選單選項。
  • list: 動態下拉選單。

    • ACOD自訂config

    額外提供常用元素檢查屬性，例如要將輸入的名稱全部改為大寫、欄位必需要填值、檢查日期格式是否正確。透過劇本設定簡化JavaScript的 撰寫加速開發進度。

    欄位檢查(jail, chk)

    元素內容檢核，在標籤加入{jail|chk(scenario1,scenario2,..)}:cmd1{;cmd2;...} 可在離開元素(jail)或是執行advConfigCheck時檢核指定劇本的元素。

  • jail:{cmd} : 游標離開元素時檢核，如果檢查不通過，就鎖住游標於輸入欄位中。
  • chk(scenario1,scenario2,..):{cmd} : 於執行advConfigCheck時檢查。

    搭配命令desc:{ColText}在檢核有誤時可於<input id="status">顯示錯誤訊息，{ColText}為欄位說明。

    • {cmd}在{jail|chk}可以為

  • string : 字串長度檢核。
  • integer : 整數值。
  • pInteger : 整數值，不可為負數。
  • decimal : 小數值。
  • pDecimal : 小數值，不可為負數。
  • chars({chr}) : 允許字元。
  • upper : 為 chars(upper) 的簡寫。
  • lower : 為 chars(lower) 的簡寫。
  • select : 表示必須選擇一個值。
  • notEmpty : 表示必須有內容。
  • date : 日期格式檢核。

    • {cmd}在{chk}可以為

  • exone(x) : 所有定義為 x 的欄位群，應恰好輸入其中一項。
  • minone(x) : 所有定義為 x 的欄位群，至少應輸入其中一項。
  • spec : 分割欄位合併。

    • 執行命令

  • advConfigCheck(Scenario,alert) : 執行chk標籤的檢查命令，檢核所有有設定為Scenario中的元素。

    Alert為True時在status顯示第一個錯誤訊息，否則為回傳一個ErrList

  • advConfigPutErrorMessage(errList) : 將errList合併為一字串。
  • advConfigShowErrorMessage(ErrStr, color) : 將錯誤字串顯示到status，color為1或-1時顯示紅色，0為正常藍色。

    • 元素長度檢查

    在{cmd}裡填入{jail|chk(s)}:string(x..y)x..y表示該元素長度需超過x但小於等於y的數值。

    預先定義網格樣式後加上ACOD標籤，可加入網格瀏覽方式或加入event觸發事件，透過to+fr功能將資料帶入或取出內容。簡化二維式資料轉入"table"元素裡，且 格內可放置input、label、select、radio等元素。

  • arge : 帶給function的參數，為一個字串

  • arge : 在網格tag所設定的參數。
  • {rowData} : 點選網格該行的資料。
  • col : 點選網格欄位名稱。
  • ndx : 網格資料行數。

  • reqAll : 回傳已存在網格內(修改後)的資料包含分頁，限查詢時已將全部查詢資料暫存於網格內。
  • reqPage : 回傳網格當前分頁的資料。
  • 未設定 : 回傳已異動網格行列。
